# SimCity End-to-End HOWTO (AXL + ENS + Ollama + Viewer)

Date: 2026-05-02

This guide shows full demo flow:
- agents walking city
- ENS lookup in viewer
- dialogues generated by local Ollama
- dialogue turn transport through AXL nodes

## 0) Prereqs

- macOS/Linux shell
- repo at `/Users/sandyagu/Desktop/simcity`
- Ollama installed
- AXL node binary present at `/Users/sandyagu/Desktop/simcity/axl/node`
- ENS subnames already minted, `axl_key` pushed to ENS text records

## 1) Terminal layout

Use 5 terminals.

- T1: Ollama
- T2: AXL nodeA
- T3: AXL nodeB
- T4: sim-city backend
- T5: viewer frontend

## 2) Start Ollama (T1)

```bash
ollama serve
```

In new shell once:

```bash
ollama pull tinyllama
ollama list
```

`tinyllama` must appear in list.

## 3) Start AXL nodes (T2, T3)

T2:

```bash
cd /Users/sandyagu/Desktop/simcity/axl_integration
npm run nodeA
```

T3:

```bash
cd /Users/sandyagu/Desktop/simcity/axl_integration
npm run nodeB
```

Optional transport sanity check:

```bash
cd /Users/sandyagu/Desktop/simcity/axl_integration
npm run phase4:run
```

Expect `node-ok`, `send`, `recv`, `reply` logs.

## 4) Start sim-city backend (T4)

```bash
cd /Users/sandyagu/Desktop/simcity/sim-city
source .venv/bin/activate
export CITYSIM_LLM_PROVIDER=ollama-openai
export CITYSIM_OLLAMA_MODEL=tinyllama
export CITYSIM_TRANSPORT=axl
export CITYSIM_TRANSPORT_REQUIRED=1
export CITYSIM_AXL_NODE_A_PORT=9002
export CITYSIM_AXL_NODE_B_PORT=9012
export CITYSIM_AXL_SEND_RETRIES=2
export CITYSIM_AXL_BACKOFF_S=0.4
python -m citysim.cli serve --n-agents 100 --max-establishments-per-kind 5 --log-level debug
```

Expected backend behavior:
- startup builds world
- dialogue worker starts
- debug logs show `axl-send` / `axl-recv`

## 5) Health + preflight checks (T4 or new shell)

```bash
curl http://127.0.0.1:8000/api/health
curl http://127.0.0.1:8000/api/health/deps
```

Deps should show `status: ok` when Ollama + both AXL nodes healthy.

Run unified preflight:

```bash
cd /Users/sandyagu/Desktop/simcity/sim-city
source .venv/bin/activate
python -m citysim.cli preflight --base http://127.0.0.1:8000 --ens a000000.simcity-7890.eth
```

Expect `[done] PASS`.

## 9) Failure map

- `model ... not found`:
  - wrong Ollama model tag
  - set `CITYSIM_OLLAMA_MODEL=tinyllama`

- `AXL node ... not reachable`:
  - nodeA/nodeB not running
  - verify ports 9002/9012

- `command not found: citysim`:
  - use module form:
  - `python -m citysim.cli ...`

- ENS lookup 404:
  - ENS not present in local DB (`ens_name` mismatch) or not generated for this run
