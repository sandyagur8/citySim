# SimCity End-to-End HOWTO (AXL + ENS + Ollama + Viewer)

Date: 2026-05-02

This guide shows full demo flow:
- agents walking city
- ENS identities synced from chain
- dialogues generated by local Ollama
- dialogue transport through AXL layer
- runtime tuning from frontend (concurrent dialogue workers + node count)
- persona edit from ENS lookup UI

## 1) Terminal layout

Use 3 terminals minimum:

- T1: Ollama
- T2: sim-city backend
- T3: viewer frontend

(Manual node terminals optional now; backend can autospawn.)

## 2) Start Ollama (T1)

```bash
ollama serve
```

One-time pull:

```bash
ollama pull tinyllama
ollama list
```

`tinyllama` must appear.

## 3) Bootstrap ENS + personas in one command (recommended)

```bash
cd /Users/sandyagu/Desktop/simcity/sim-city
source .venv/bin/activate
python -m citysim.cli bootstrap-ens --n-agents 100 --grid-size 80 --seed 42 --batch-size 20
```

This runs:
1. persona generation
2. wallet backfill
3. axl key backfill
4. ENS mint in batches
5. ENS text push (parallel)
6. chain sync -> local DB status

If already minted before:

```bash
python -m citysim.cli sync-ens-status --limit 1000 --concurrency 20
```

## 4) Start backend with AXL autospawn (T2)

```bash
cd /Users/sandyagu/Desktop/simcity/sim-city
source .venv/bin/activate
export CITYSIM_LLM_PROVIDER=ollama-openai
export CITYSIM_OLLAMA_MODEL=tinyllama
export CITYSIM_TRANSPORT=axl
export CITYSIM_TRANSPORT_REQUIRED=1
export CITYSIM_AXL_AUTOSPAWN=1
export CITYSIM_AXL_NODE_COUNT=2
export CITYSIM_AXL_SEND_RETRIES=2
export CITYSIM_AXL_BACKOFF_S=0.4
export CITYSIM_AXL_NODE_A_PEER_ID=1fd26e5f5247f758e5204b281247036daf95d768b615dccc57de5859078a15b3
export CITYSIM_AXL_NODE_B_PEER_ID=63d1c9d07ab3b30c5aaf2c369ee00ae1b264c63194856401049570c1562798cc
python -m citysim.cli serve --n-agents 100 --max-establishments-per-kind 5 --log-level debug
```

Expected backend behavior:
- autospawn enabled -> AXL nodes started by backend
- worker pool started
- logs show `axl-send` / `axl-recv`

When backend exits, spawned AXL nodes are terminated automatically.

## 10) Failure map

- `model ... not found`
  - set `CITYSIM_OLLAMA_MODEL=tinyllama`

- `AXL receive timeout ...`
  - node down, bad peer-id env, or node-count > available configs

- all ENS pending
  - local DB not synced
  - run `citysim sync-ens-status`

- `command not found: citysim`
  - use module form: `python -m citysim.cli ...`
